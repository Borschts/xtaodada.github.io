正式完成从Wordpress迁移至Hexo

2019年08月02日 582 字 大概 2 分钟 日常 hexo 公告 迁移 通知

为什么改变?

wordpress 由于日度使用变得越来越臃肿,很多朋友给我说 该改一改了 但是一直都没有找到机会去修改这个问题.我先说说在使用中的一些问题.

  1. 在日常写作中,我发现特别是 技术 类文章在插入代码时,你就需要去安装代码高亮插件,然后通过短代码进行框选,在一篇文章需要文章和代码间断时,这个步骤就会变得十分枯燥、乏味.
  2. 在写完一篇文章后,想要备份到word或者txt时,就会出现一大短代码.这对文章的阅读极其不友好.极大的耗费时间和精力
  3. 切换主题的不方便.在用一个主题时,这个主题的教程有可能会写上一大堆短代码,然后你也就用了,但是在你切换主题时,这些短代码并不会自动改变,而且有些功能新主题没有,这就出现了又一个耗费时间和精力的工程

其实以上问题都微不足道,因为WP有很多的插件进行弥补和替换.但是在github这些readme文档中需要使用 Markdown .虽然WP在新版本中加入了 Markdown 的支持,但写作体验还是感觉不行

为什么要选择Hexo

  1. 接触 Git 和 Markdown 得越来越多,而且,平常很少有时间去水博客.如果使用 Hexo 水博客,那这个过程就会变得十分简单,生成相应的静态文件,再一键部署。
  2. Hexo是一个快速、简洁且高效的博客框架.所有的文章都是以Markdown的文本格式保存,这样把文章渲染成html文档. 十分高逼格
  3. 全命令行操作,简单粗暴

迁移后感谢

2019-08-02 12:00:00 正式完成从 WordPress 迁移至 Hexo 的工作.
从 2018/02 到现在 2019/08 也已经有段时间了, 在这期间, 陆陆续续写了30篇文章
看着与以往不同的博客页面和简洁的样式,突然觉得自己的迁移好像还是不错的.
